### NEO4J MODELING CONVENTIONS & REASONING
- Labels (Nodes): Represent generic entities/nouns. Must be singular and PascalCase. 
  *Example of Identifying a New Label:* If the current ontology has a `Patient` and `Procedure`, and the new question asks about "Medical Devices" used, "Medical Device" is a distinct entity (noun) that requires its own node. Add a new `MedicalDevice` label.
- RelationshipTypes (Edges): Represent verbs/actions. Must be UPPER_SNAKE_CASE. 
  *CRITICAL RULE:* NEVER embed node Label names into the relationship name. Keep relationships abstract and reusable. 
  *Example:* Use `USED` (not `USED_DEVICE`), use `PRESCRIBED` (not `PRESCRIBED_MEDICATION`).
- Properties: Represent attributes. Must be lowerCamelCase (e.g., `modelNumber`, `severityLevel`). Valid types are: "string", "integer", "float", "boolean", "date", "list".
- SchemaMap: Defines the directional connection (start_label -> type -> end_label).
